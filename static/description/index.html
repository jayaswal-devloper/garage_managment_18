Configuration Flow

Step-by-step configuration of course, semester, and academic term.

Education Configuration

Admin first creates the education details (like 10th, 12th, etc.) before any student registration. Then create a year.

Course & Term Assignment

When you select a course and enter the semester, the system automatically creates the semester linked to that course without affecting subjects.

Eligibility for the Course

The requirements you need to fulfill to take this course.

Academic Term Configuration

The academic term name includes the course, semester, and academic year (e.g., B.COM - SEM-1 - 2025-26).

System Flow

Step-by-step workflow from registration to portal access

Registration

Student fills out the registration form on the website. Email is unique.

Requested Data

After form submission, the application is marked as ‘Applied’.

Portal/Admin User

Once approved, a user is created: students as portal users, and faculty/admin as internal users.

Contact Type

A selection field is added to define user roles: Admin, Teacher, Student, and Staff. This field helps categorize contacts and manage user access accordingly.
